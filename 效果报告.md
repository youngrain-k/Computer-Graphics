## 一、项目架构
该项目采用模块化分层设计，将代码按功能拆分为 3 个核心文件，结构清晰且职责明确：

config.py	集中管理所有可配置参数（物理系统、渲染系统），便于调试和参数调整，无需修改核心逻辑

physics.py	定义粒子物理规则（初始化、受力计算、边界碰撞），基于 Taichi 实现 GPU 并行计算

main.py	程序入口，负责 Taichi 初始化、GUI 窗口创建、主循环渲染及物理逻辑驱动

## 二、代码逻辑
整体执行流程遵循「初始化 → 主循环（物理计算 + 渲染）」的经典图形学逻辑，核心步骤如下：

环境初始化：main.py 中先执行 ti.init(arch=ti.gpu) 接管 GPU，调用 init_particles() 为所有粒子分配随机初始坐标（速度初始化为 0）。

主循环驱动：

读取鼠标实时坐标（归一化到 0-1 范围）；

调用 update_particles() 由 GPU 并行计算每个粒子的物理状态：

计算粒子到鼠标的方向和距离，施加引力（距离过近时忽略，避免加速度过大）；

叠加空气阻力（速度乘以阻力系数），更新粒子位置；

检测边界碰撞，超出窗口范围时反弹（速度乘以反弹系数，损耗能量）；

从显存读取粒子坐标数据，通过 gui.circles() 绘制粒子，刷新窗口。

终止条件：当关闭 GUI 窗口时，主循环结束，程序退出。

## 三、实现功能
该项目基于 Taichi 框架实现了交互式引力粒子群效果，核心功能包括：

GPU 并行计算：利用 Taichi 的 ti.kernel 装饰器，让1万+粒子的物理计算在 GPU 上并行执行，保证高帧率运行；

鼠标交互引力：粒子会向鼠标光标位置产生引力运动，引力强度可通过 GRAVITY_STRENGTH 调整；

物理规则模拟：

空气阻力：粒子速度随时间衰减（DRAG_COEF 控制衰减程度）；

边界反弹：粒子碰到窗口边缘时反弹，且反弹后能量有损耗（BOUNCE_COEF 控制）；

可视化渲染：以天蓝色圆形粒子渲染，支持自定义窗口分辨率、粒子大小和颜色。

## 总结
项目核心是利用 Taichi 实现 GPU 并行加速，解决大量粒子的实时物理计算性能问题；

模块化设计让参数调整和功能扩展更便捷（如修改粒子数量、引力强度仅需改 config.py）；

核心逻辑围绕「鼠标引力 + 物理规则 + 实时渲染」，实现了交互式的粒子群视觉效果。
